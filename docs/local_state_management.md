# Tifossi Local State Management Specification

This document details the local state management strategy for the Tifossi Expo application, focusing on client-side state using Zustand for global state and React's built-in state hooks for component-level UI state. This approach uses mock data initially and excludes backend interactions.

## 1. Architecture Overview

Local state is managed using a two-pronged approach:

1.  **Global Client State (Zustand):** Manages state that needs to be shared across multiple components or screens, independent of the component tree structure. Examples include the shopping cart, user favorites, and potentially UI theme preferences.
2.  **Local Component State (React Hooks):** Manages state specific to a single component or a closely related group of components. This includes UI state like form inputs, modal visibility, accordion expansion, temporary selections, etc.

## 2. Global Client State Management (Zustand)

-   **Responsibility:** Managing application-wide state accessible from anywhere.
-   **Library:** `zustand`
-   **Implementation:**
    -   State is organized into focused stores (slices) located in `app/_stores/` (e.g., `cartStore.ts`, `favoritesStore.ts`).
    -   Components access and update state via hooks generated by Zustand stores (e.g., `useCartStore(state => state.items)`).
-   **Persistence:**
    -   Critical client state (Cart, Favorites) will be persisted locally using `react-native-mmkv` for fast, synchronous storage access.
    -   Zustand's `persist` middleware is used with a custom MMKV storage adapter.
-   **Key Stores:**

    ### a) Cart Store (`app/_stores/cartStore.ts`)

    Manages the items in the user's shopping cart.

    ```typescript
    // In app/_stores/cartStore.ts
    import { create } from 'zustand';
    import { persist, createJSONStorage } from 'zustand/middleware';
    import { MMKV } from 'react-native-mmkv';
    import { Product } from '../_types/product'; // Assuming Product type exists

    // Setup MMKV storage
    const storage = new MMKV();
    const mmkvStorage = createJSONStorage(() => ({
      getItem: (name) => storage.getString(name) ?? null,
      setItem: (name, value) => storage.set(name, value),
      removeItem: (name) => storage.delete(name),
    }));

    export interface CartItem {
      productId: string;
      quantity: number;
      color?: string; // hex value or name
      size?: string; // e.g., 'S', 'M', 'L'
      // Add other variant details if necessary
    }

    interface CartState {
      items: CartItem[];
      addItem: (item: CartItem) => void;
      updateItemQuantity: (productId: string, color: string | undefined, size: string | undefined, newQuantity: number) => void;
      removeItem: (productId: string, color: string | undefined, size: string | undefined) => void;
      clearCart: () => void;
      getItemQuantity: (productId: string, color: string | undefined, size: string | undefined) => number;
    }

    export const useCartStore = create<CartState>()(
      persist(
        (set, get) => ({
          items: [],

          addItem: (itemToAdd) => {
            set((state) => {
              const existingItemIndex = state.items.findIndex(
                (item) =>
                  item.productId === itemToAdd.productId &&
                  item.color === itemToAdd.color &&
                  item.size === itemToAdd.size
              );

              if (existingItemIndex > -1) {
                // Item exists, update quantity
                const updatedItems = [...state.items];
                updatedItems[existingItemIndex] = {
                  ...updatedItems[existingItemIndex],
                  quantity: updatedItems[existingItemIndex].quantity + itemToAdd.quantity,
                };
                return { items: updatedItems };
              } else {
                // Add new item
                return { items: [...state.items, itemToAdd] };
              }
            });
            // NOTE: Backend sync logic is omitted in this local-only spec.
          },

          updateItemQuantity: (productId, color, size, newQuantity) => {
            set((state) => {
              if (newQuantity <= 0) {
                // If quantity is zero or less, remove the item
                return {
                  items: state.items.filter(
                    (item) =>
                      !(item.productId === productId && item.color === color && item.size === size)
                  ),
                };
              } else {
                // Update quantity for the specific item variant
                return {
                  items: state.items.map((item) =>
                    item.productId === productId && item.color === color && item.size === size
                      ? { ...item, quantity: newQuantity }
                      : item
                  ),
                };
              }
            });
             // NOTE: Backend sync logic is omitted.
          },

          removeItem: (productId, color, size) => {
            set((state) => ({
              items: state.items.filter(
                (item) =>
                  !(item.productId === productId && item.color === color && item.size === size)
              ),
            }));
             // NOTE: Backend sync logic is omitted.
          },

          clearCart: () => {
            set({ items: [] });
             // NOTE: Backend sync logic is omitted.
          },

          getItemQuantity: (productId, color, size) => {
            const item = get().items.find(
              (i) => i.productId === productId && i.color === color && i.size === size
            );
            return item ? item.quantity : 0;
          },
        }),
        {
          name: 'tifossi-cart-local', // Unique name for local storage
          storage: mmkvStorage,
        }
      )
    );
    ```

    ### b) Favorites Store (`app/_stores/favoritesStore.ts`)

    Manages the list of user's favorite product IDs.

    ```typescript
    // In app/_stores/favoritesStore.ts
    import { create } from 'zustand';
    import { persist, createJSONStorage } from 'zustand/middleware';
    import { MMKV } from 'react-native-mmkv';

    // Setup MMKV storage (can share instance or use a new one)
    const storage = new MMKV({ id: 'favorites-storage' });
    const mmkvStorage = createJSONStorage(() => ({
      getItem: (name) => storage.getString(name) ?? null,
      setItem: (name, value) => storage.set(name, value),
      removeItem: (name) => storage.delete(name),
    }));

    interface FavoritesState {
      productIds: string[];
      addFavorite: (productId: string) => void;
      removeFavorite: (productId: string) => void;
      toggleFavorite: (productId: string) => void;
      isFavorite: (productId: string) => boolean;
    }

    export const useFavoritesStore = create<FavoritesState>()(
      persist(
        (set, get) => ({
          productIds: [],

          addFavorite: (productId) => {
            set((state) => {
              if (!state.productIds.includes(productId)) {
                return { productIds: [...state.productIds, productId] };
              }
              return state; // No change if already present
            });
            // NOTE: Backend sync logic omitted.
          },

          removeFavorite: (productId) => {
            set((state) => ({
              productIds: state.productIds.filter((id) => id !== productId),
            }));
            // NOTE: Backend sync logic omitted.
          },

          toggleFavorite: (productId) => {
            const { productIds, addFavorite, removeFavorite } = get();
            if (productIds.includes(productId)) {
              removeFavorite(productId);
            } else {
              addFavorite(productId);
            }
          },

          isFavorite: (productId) => {
            return get().productIds.includes(productId);
          },
        }),
        {
          name: 'tifossi-favorites-local', // Unique name
          storage: mmkvStorage,
        }
      )
    );
    ```

## 3. Local Component State (React Hooks)

-   **Responsibility:** Managing state that is specific to a component's rendering logic, UI interactions, or temporary data.
-   **Libraries:** React (`useState`, `useReducer`, `useRef`)
-   **Implementation:** Use standard React state hooks within functional components.
-   **Examples:**
    -   `useState`: Managing input field values, checkbox states, modal visibility, selected tabs, loading states within a component, animation states.
    -   `useReducer`: Managing complex state logic within a component, especially when the next state depends on the previous one or involves multiple sub-values (e.g., a complex form with validation).
    -   `useRef`: Holding mutable values that don't trigger re-renders (e.g., references to DOM elements, timers, previous state values).

```tsx
// Example Component using both Zustand and Local State
import React, { useState } from 'react';
import { View, Button, Text, TextInput } from 'react-native';
import { useCartStore } from '../../app/_stores/cartStore'; // Adjust path
import { Product } from '../../app/_types/product'; // Adjust path

interface ProductDetailProps {
  product: Product; // Assume product is passed as a prop
}

export default function ProductDetail({ product }: ProductDetailProps) {
  // Local state for UI interaction (e.g., selected color/size)
  const [selectedColor, setSelectedColor] = useState<string | undefined>(
    product.colors?.[0]?.hex // Default to first color hex
  );
  const [selectedSize, setSelectedSize] = useState<string | undefined>(
    product.sizes?.find(s => s.available)?.value // Default to first available size
  );
  const [quantity, setQuantity] = useState<number>(1); // Local quantity input

  // Global state from Zustand
  const addItemToCart = useCartStore((state) => state.addItem);
  const cartQuantity = useCartStore((state) =>
    state.getItemQuantity(product.id, selectedColor, selectedSize)
  );

  const handleAddToCart = () => {
    if (quantity > 0) {
       addItemToCart({
         productId: product.id,
         quantity: quantity,
         color: selectedColor,
         size: selectedSize,
       });
      // Optionally reset local quantity or give feedback
      // setQuantity(1); 
    }
  };

  return (
    <View>
      <Text>{product.title}</Text>
      {/* UI for color selection (updates selectedColor state) */}
      {/* UI for size selection (updates selectedSize state) */}

      <TextInput
        value={String(quantity)}
        onChangeText={(text) => setQuantity(parseInt(text, 10) || 1)}
        keyboardType="numeric"
      />

      <Button title="Add to Cart" onPress={handleAddToCart} />

      <Text>In Cart: {cartQuantity}</Text>
    </View>
  );
}
```

## 4. Data Flow and Integration

-   Zustand stores (`cartStore`, `favoritesStore`) hold the persistent global client state (cart items, favorite IDs).
-   Components subscribe to relevant parts of the Zustand stores using selector hooks.
-   Components use local `useState` or `useReducer` for their internal UI logic (e.g., form inputs, selections, modal visibility).
-   When a user interacts (e.g., adds to cart), the component updates its local state (if necessary) and calls action functions from the relevant Zustand store (e.g., `addItemToCart`).
-   Zustand updates its state, and the `persist` middleware automatically saves it to MMKV storage.
-   Components subscribed to the changed part of the Zustand store re-render.
-   Initial data (like product details) is sourced from mock data (`app/_data/products.ts`) passed via props or fetched within the component/screen initially.

## 5. Conclusion

This local state management strategy provides a clear separation between global, persistent client state (managed by Zustand and persisted with MMKV) and ephemeral component-level UI state (managed by React hooks). It leverages mock data for initial development and testing, establishing the core state management patterns before integrating backend services.
